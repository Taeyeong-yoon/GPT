# 네코짱 테스트 — 작업 기록 (2026-06-08)

## 1. SJPT 2/3부 문제 이미지 미표시 수정
- **증상**: SJPT 2부·3부 문제에서 이미지가 화면에 나오지 않음
- **원인 1**: `Exam.jsx`에서 `{part === 2 && q.imageUrl && <img/>}` 조건으로 3부 이미지를 아예 렌더링 대상에서 제외
- **원인 2 (근본 원인)**: 이미지 URL이 `drive.google.com/uc?export=view`였는데, 이 주소가 `drive.usercontent.google.com`으로 리다이렉트되며 `Cross-Origin-Resource-Policy: same-site` 헤더를 반환 → 브라우저가 외부 도메인의 `<img>` 로딩을 조용히 차단(curl로는 헤더를 무시해 재현 안 됨)
- **조치**:
  - 렌더링 조건을 `{q.imageUrl && <img/>}`로 수정 (2부·3부 모두 표시)
  - 이미지 CDN을 CORS 허용 도메인 `https://lh3.googleusercontent.com/d/{id}=w1000`으로 교체 (`api/sjpt-questions.js`)
- 커밋: `af2ffe9`, `1154deb`

## 2. JLPT 청해 음성 재생 불가 수정
- **증상**: JLPT 청해 문제에서 음성 재생 버튼을 눌러도 "재생 실패" 표시
- **원인**: `handlePlay`가 `useCallback`으로 캐싱되는 시점이 `loading`/`error` 조기 반환 구간이라, 아직 초기화되지 않은 `item` 변수를 클로저로 참조 → `ReferenceError: Cannot access 'item' before initialization`(TDZ 오류)가 발생해 catch 블록에서 "재생 실패"로 표시됨
- **조치**: `handlePlay(text, subType)`처럼 필요한 값을 인자로 직접 전달받도록 수정해 클로저 문제 제거
- 커밋: `93c2744`

## 3. SJPT 부별 문제 중복 출제 방지
- **증상**: SJPT 1~7부 응시 시 같은 부에서 동일한 문제가 2번 출제되는 사례 발생 (예: 3부)
- **원인**: 구글 시트의 문제 풀에 동일 텍스트의 문항이 난이도/이미지 변형별로 최대 12회까지 중복 저장되어 있어, 무작위 추출 시 같은 문제가 두 번 뽑힐 수 있었음
- **조치**: `dedupeByText()`를 추가해 부별 문제 풀에서 텍스트 기준 중복을 사전에 제거한 뒤 무작위 추출 → 구조적으로 중복이 불가능하도록 변경
- 커밋: `9d9fc22`

- `src/features/sjpt/Setup.jsx`에 적용, 관련 CSS(`.cat-icon`, `.cat-icon__accent`) 추가
